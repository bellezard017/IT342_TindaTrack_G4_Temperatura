Refactoring Summary

The backend and web frontend were refactored toward Vertical Slice Architecture by grouping sales and store-management behavior by feature instead of keeping those flows only in technical-layer folders. Existing auth, user, security, DTO, validation, and shared service infrastructure remain in their current package areas because they are cross-cutting support code used by more than one slice.

  • Moved sales backend API, service, repository, and entity into com.tindatrack.backend.features.sales.
  • Moved store backend API, service, repository, and entity into com.tindatrack.backend.features.stores.
  • Updated backend shared services such as auth, Google OAuth, sale factory, observers, and notifications to import the new slice classes.
  • Moved web sales records view and sales API client into web/src/features/sales.
  • Moved web store management view and store API client into web/src/features/stores.
  • Updated routes and page imports so existing user flows point to the new feature modules.
  • Fixed the Windows Maven wrapper script so regression tests can run through mvnw.cmd.
